## 二、問題與痛點
直播內容爆炸成長，但**精華短片仍高度依賴人工剪輯**：
- 一場 2–4 小時直播，剪輯師需**完整回看**才能定位精彩片段；
- 每支短片需手動裁切、轉直式 9:16、上字幕、想標題與鉤子；
- 還要為 TikTok / Reels / Shorts **各自重做**版型；
- 平台內容審核與合規則常被忽略，帶來下架與商譽風險。

結果：**高光內容的產出速度，遠遠跟不上直播產生的速度**，變現與二次流量被浪費。

## 三、解決方案概述
SparkReel 是一套**多模態直播高光偵測 × AI 自動剪輯**代理系統：

1. **辨識高光**：同時分析四大模態訊號並融合評分 —
   - 🔊 **語音情緒**：音量爆點（歡呼/笑聲）、語音激動度（頻譜）
   - 💬 **彈幕情感**：洗版爆量偵測、群眾情緒強度（中英俚語詞典 / Bedrock）
   - 🎞️ **視覺動態**：畫面動態量、場景切換、特寫/表情反應（OpenCV / Rekognition）
   - 🗣️ **逐字稿關鍵語**：「這波太神」「五殺」等高能語（Transcribe → 詞典 / Bedrock）
2. **自動剪輯**：幀級裁切、9:16 重構圖、燒錄字幕、限時開場鉤子、精華合輯敘事拼接。
3. **多風格產出**：一鍵輸出 TikTok / IG Reels / YouTube Shorts 三種平台原生版型。
4. **審核合規**：三模態內容審核 + 人工複審路由，安全上片。
5. **可解釋**：每個高光都附「為什麼」與證據，讓創作者信任並微調。

> **設計關鍵**：本地備援 + AWS 雲端雙模式。零憑證即可完整運行，展示與落地無縫銜接。

## 四、涵蓋的多模態訊號
共 9 條正規化訊號，加權融合成單一高光分數曲線（見 [`架構設計.md`](架構設計.md)）。
每條訊號皆有「本地引擎」與「AWS 轉接層」兩種實作，可獨立於 `config/default.yaml` 切換。

## 五、與現有工具的差異
| | 一般自動剪輯工具 | SparkReel |
|---|---|---|
| 偵測依據 | 多半僅音量或聊天單一維度 | **四模態九訊號融合** |
| 可解釋性 | 黑箱 | **逐高光歸因 + 證據** |
| 平台適配 | 單一輸出 | **三平台原生版型自動化** |
| 合規 | 缺乏 | **三模態審核 + 人工複審** |
| 部署 | 綁定雲端 | **本地/雲端雙模、優雅降級** |
