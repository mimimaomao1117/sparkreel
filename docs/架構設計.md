# 模組設計

## (1) 智慧內容分析：高光偵測邏輯與多模態融合策略

### 1.1 分析網格
以固定時間網格（預設 2 Hz，每 0.5 秒一格）對齊所有訊號，使異質模態可加權相加。

### 1.2 九條訊號與正規化
每條訊號抽取後以 **百分位截斷正規化**（5%–95%）壓到 `0..1`，抵抗單一離群值：

```
audio_excitement  = norm(RMS 能量) 0.55 + norm(超越移動基線的爆量) 0.7
audio_emotion     = (norm(頻譜質心) + norm(頻譜通量)) × 語音門控
speech_keyword    = 逐字稿命中高能詞（詞典/Bedrock）的強度
speech_sentiment  = |情感極性|（正負皆為情緒高峰）
chat_volume       = norm(訊息速率) 0.5 + norm(洗版爆量) 0.7
chat_sentiment    = 群眾情緒強度 × (0.5 + 0.5×爆量)
visual_motion     = norm(相鄰幀差)
visual_scene      = norm(色彩直方圖散度)  # 場景切換/重播
visual_face       = 人臉面積比 / Rekognition 表情信心
```

### 1.3 多模態融合策略
```
fused(t) = Σ wᵢ · sᵢ(t) / Σ wᵢ      # 僅計入「有效」訊號（避免死模態稀釋）
fused    = moving_average(fused, 平滑視窗)
```
權重 `wᵢ` 於 `config/default.yaml` 可調（彈幕爆量預設最高 1.2，因其為觀眾即時投票）。

### 1.4 峰值偵測與高光視窗
1. 找出 `fused ≥ 門檻(0.55)` 的局部極大值；
2. 依分數由高至低貪婪選取，強制**最小間隔（8s）**，取上限（8 個）；
3. 每個峰值向左右擴張至低於次門檻，套用前後鋪陳 padding，箝制於 `[最短, 最長]`；
4. 相鄰視窗重疊時裁切，避免重複素材。

### 1.5 可解釋歸因
每個高光輸出：主導訊號（前三名貢獻）、關鍵語、逐字稿摘錄、熱門彈幕、峰值分數。
→ 讓創作者理解「為什麼這段被選中」，並可回饋調權。

## (2) AI 自動剪輯引擎：裁切拼接與敘事結構生成

### 2.1 幀級精準裁切
以 ffmpeg **輸入端 seek + PTS 重置** 保證裁切點與字幕時間軸幀級對齊（跨 GOP 亦準確）。

### 2.2 版型重構圖（Reframe）
`crop → 中心裁切至 9:16 → scale 1080×1920`，同時支援橫向與直向來源。

### 2.3 字幕與鉤子
- **字幕**：由逐字稿產生樣式化 ASS（libass 燒錄），CJK 字型（WenQuanYi Zen Hei），
  平台安全區位（`bold_center` / `bold_top`）；
- **開場鉤子**：限時（前 2.6 秒）drawtext 疊字，抓住前 3 秒留存率。

### 2.4 敘事結構生成
- **單片敘事**：鋪陳 → 爆點 → 反應（由高光視窗與 padding 界定）；
- **精華合輯**：`hook（最佳片段冷開場）→ 時序推進 → payoff` 排序拼接；
- **文案**：標題 / 鉤子 / 主題標籤，由樣板（本地）或 **Bedrock（Claude）** 依高光證據生成。

### 2.5 穩健降級
剪輯採「濾鏡降級階梯」：完整（重構圖＋字幕＋鉤子）失敗 → 去鉤子 → 去字幕 → 純重構圖，
**保證每支短片都能產出**。

## (3) 多風格內容產出：系統功能設計與平台適配

### 3.1 平台預設（`styles/platforms.py` + `config`）
| 平台 | 版型 | 時長上限 | 字幕位置 | 鉤子語氣 |
|---|---|---|---|---|
| TikTok | 9:16 1080×1920 | 60s | 中央粗體 | 直球爆點 |
| IG Reels | 9:16 1080×1920 | 90s | 中央粗體 | 美感敘事 |
| YouTube Shorts | 9:16 1080×1920 | 60s | 頂部（讓出底部 UI） | 資訊導向 |

### 3.2 一源多出
單一高光 → 迴圈套用各平台預設 → 產生獨立變體（不同字幕樣式、鉤子、時長箝制），
每支附縮圖、時長、尺寸、檔案大小與**預測品質分**。

### 3.3 可擴充
新增平台只需在 `config` 增加一組預設；新增訊號只需實作一個抽取器並註冊到 `signals/EXTRACTORS`。
